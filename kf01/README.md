# kf01 往日重现 2.0 项目大纲

更新时间：2026-06-09

## 产品定位

《往日重现 (Yesterday Once More)》是家庭影像智能策展与 Android TV 展播系统。系统目标不是简单播放 NAS 照片，而是把本地目录、飞牛 NAS 相册和后续私有源统一入池，经过转码、AI 评分、AI 文案、排版设计和质量优先推送后，在家庭电视上呈现每日精选记忆。

## 2.0 业务主线

```text
物理源：飞牛 NAS / 本地目录
-> 按需或扫描入池
-> 异步三规格派生图：300px 管控端缩略图 + 720p AI 分析图 + 4K WebP 放映图
-> 虚拟播放相册映射和策展分拣
-> 每日限量 AI 漏斗：单次 Vision 优先，必要时拆分评分和排版
-> 广域网账号登录和 JWT 鉴权
-> 质量优先下发：新高分图不足时用历史经典图补齐
-> Android TV 边缘缓存和沉浸放映
```

## 当前工程迁移判断

- 已完成能力继续兼容：照片中心、本地扫描、飞牛相册源、共享相册、播放相册、快速分拣台、相册级 AI 开关、AI 设置中心、TV 账号密码登录。
- 当前 NestJS + SQLite 后端作为 2.0 验证桥接层继续保留，用于快速跑通业务闭环；目标架构迁移到 Python FastAPI + PostgreSQL + Redis/Celery/APScheduler 时，不删除已验证的 API 契约和业务规则。
- 飞牛“扫描”只同步相册清单；飞牛照片在播放相册查看、AI 生成、TV 推送等需要时按需读取并持久化，避免全量枚举所有 NAS 照片。
- Android TV 不直接登录飞牛，也不保存飞牛账号；TV 只登录往日重现后端地址，消费后端签发的内容流和媒体 URL。

## 2.0 模块边界

1. 核心中枢与数据引擎：照片指纹、EXIF、三规格派生图、统一 Vision AI 队列、质量优先推送、JWT。
2. Vben 管控端：照片中心、播放相册、快速分拣、AI 设置、AI 调度、往日画廊、设备配置。
3. Android TV 放映端：广域网登录、每日静态流同步、4K WebP 缓存清理、Ken Burns 展示、AI 文案安全区。
4. 部署集成：Docker Compose、PostgreSQL、Redis、物理目录挂载、APK 更新脚本。

## 近期落地顺序

1. 固化 kf01 和 OpenAPI 契约。
2. 完成飞牛播放相册照片按需导入，并持久化 AI 状态。
3. 用现有 NestJS 后端补齐 2.0 API 行为测试。
4. 做三规格派生图模型和衍生资产字段，分别服务管控端缩略图、AI 省流量分析和 TV 4K WebP 放映。
5. 接入真实 Vision API，优先用单次请求完成评分、点评、分类和 TV 设计；效果不稳时再拆成两阶段提示词。
6. 生成每日 `daily_themes.json`，Android TV 拉取并展示。
