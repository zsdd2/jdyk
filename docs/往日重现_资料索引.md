## 更新规则

1. 业务主线变更：先改 `kf01/README.md`，再改详细计划。
2. 开发原则变更：只改 `kf01/DEVELOPMENT_CONSTITUTION.md`。
3. API 字段变更：先改 `kf01/api-contracts/openapi.yaml`，再改代码和测试。
4. AI 返回格式、存储、展示流程变更：只改 `kf01/AI_FEEDBACK_FLOW.md`，其他文档只链接或摘要。
5. 每轮开发结束：只在 `PROJECT_DEVELOPMENT.md` 更新当前状态、问题和下一步，历史细节归档到 `docs/archive/`。
